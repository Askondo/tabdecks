# 2026-06-13 — Initial build

Full MVP implemented and committed in 7 phases (see git log b10ac9c..HEAD):
scaffold (WXT+Svelte 5), dual-tab capture, guarded AudioEngine + crossfader/master,
LR4 EQ with kills, FX framework + echo, transport worklet (live/brake/stutter),
timeshift + track mode + waveform, polish (meters/shortcuts/settings/zip).

## Verified knowledge captured

- **BiquadFilter Q is in dB for lowpass/highpass** (Butterworth = -3.0103, not 0.7071).
  Caught by render tests: +7.4 dB crossover overshoot. → docs/api/webaudio_notes.md
- Render tests run via **node-web-audio-api** in plain vitest node — no browser needed
  (replaced the planned vitest-browser-mode + Playwright approach; same coverage, lighter).
- WXT unlisted-script wrapper executes `main()` at module evaluation → safe for
  `registerProcessor` in AudioWorklet modules.
- Root tsconfig `include` must list `.wxt/wxt.d.ts` or `#imports` types vanish.
- **Branded Chrome stable ignores `--load-extension` since v137** — browser opens, extension
  silently absent (zero `--extension-process` renderers). Use Chrome for Testing for the dev
  launcher (`npx @puppeteer/browsers install chrome@stable`); launcher now warns on misconfig.
